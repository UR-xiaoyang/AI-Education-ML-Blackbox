### P0 问题（严重教学缺陷 — 共 5 个）

#### 1. YOLO Lab 使用假的预测动画（欺骗性设计）
**严重程度**: Critical  
**文件**: `src/labs/YOLOLab.jsx`  
**问题**: YOLO Lab 的预测结果来自硬编码的 `PREDICTION_TEMPLATES` 数组，而非真实神经网络输出。学生以为看到了真实的模型训练，实际上是预设动画。

**影响**: 学生对目标检测模型如何学习的理解是根本错误的。  
**修复方案**: 在实验标题区域和所有检测画布上添加明显的"演示动画"免责声明。  
**修改文件**: `src/labs/YOLOLab.jsx`
- 标题区域：添加"📺 注意：预测框变化是基于模板插值的教学演示动画，非真实神经网络输出。"提示框
- 所有检测画布：左上角添加"📺 演示动画"角标

**状态**: ✅ 已修复 — 添加了明确的演示动画免责声明

---

#### 2. LLM Lab 术语错误 — "预训练"主题实际上是"微调"
**严重程度**: Critical  
**文件**: `src/store/scenarioConfig.ts` (llm_step_2_theme_selection)  
**问题**: 场景配置中将"问答训练"、"故事生成"、"技术文档"称为"预训练主题"。但预训练（Pre-training）是在无标注大语料上学习语言模型，下游任务的微调（Fine-tuning）才是用特定主题数据。

**修复方案**: 在引导文案中添加解释，说明这里模拟的是微调而非预训练。  
**状态**: ✅ 已修复

---

#### 3. 核心概念首次出现时无解释
**严重程度**: High  
**修复方案**: 在引导文案中添加通俗解释（使用💡符号）。  
**状态**: ✅ 已修复

| 术语 | 改进 |
|------|------|
| XOR | 添加"什么是XOR？简单说就是：'两个输入相同则输出0，不同则输出1'——这种模式用一条直线永远分不开！" |
| 学习率 | 添加"把梯度下降想象成下山：学习率就是你每一步迈多远——步子太大可能越过山谷（震荡/爆炸），步子太小则下山太慢" |
| 神经元 | 添加"什么是神经元？把它想象成一个小计算器：它接收几个输入数字，分别乘以不同的'权重'后相加，最后通过一个'激活函数'输出一个结果" |
| IoU | 添加"💡 IoU = Intersection over Union，即'预测框'与'真实框'的交集面积除以并集面积，范围 0~1，越接近 1 表示预测越准" |

#### 5. 学生操作错误时缺乏即时反馈
**严重程度**: High  
**文件**: `src/labs/LinearRegressionLab.jsx`  
**修复方案**: 在线性回归 Lab 的 stepTitle/stepDesc 区域添加数据质量实时检测。

**检测逻辑**:
- 仅 1 个点：显示红色警告"⚠️ 仅 1 个点，无法计算误差！"
- 仅 2 个点：显示黄色提示"💡 2 个点太少，模型可能欠拟合，建议添加更多点。"
- X 值范围太小：显示红色警告"⚠️ X 值范围太小（集中在同一位置），模型无法学到有效模式！"
- Y 值几乎相同：显示红色警告"⚠️ Y 值几乎相同，所有点在同一水平线上！"

**状态**: ✅ 已修复 — 线性回归 Lab 已添加实时数据质量检测

#### 8. Spotlight 气泡位置算法偶发错误
**严重程度**: Medium  
**文件**: `src/components/SpotlightOverlay.jsx`  
**问题**: 空间不足时 fallback 到角落（top: 20, left: 20），导致气泡被遮挡。

**修复方案**: 将空间不足时的 fallback 从角落改为始终居中显示。  
**修改**:
```javascript
// 修改前
} else {
  top = 20;
  left = 20;
  transform = 'translate(0, 0)';
}

// 修改后
} else {
  // 空间不足，优先居中显示，避免放角落被遮挡
  top = '50%';
  left = '50%';
  transform = 'translate(-50%, -50%)';
  return;
}
```

**状态**: ✅ 已修复
